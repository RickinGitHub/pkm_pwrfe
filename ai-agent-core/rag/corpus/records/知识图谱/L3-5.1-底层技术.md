## 🔷 5.1.1 Linux V4L2 架构 `[精华]`

| 颗粒度 | 细化内容 |
|--------|----------|
| **文件** | `./[精华]Linux-V4L2-架构解析与图示.md` |
| **▸ 四层架构·逐层展开** | ① **用户空间**：libv4l/FFmpeg/GStreamer——应用层通过ioctl与内核交互。**关键接口**：VIDIOC_REQBUFS（请求缓冲区）/VIDIOC_QBUF（入队）/VIDIOC_DQBUF（出队）/VIDIOC_STREAMON（启动流）② **V4L2核心层**：ioctl调度+VB2（Videobuf2）缓冲区管理——整个V4L2的"交通枢纽"。**核心职责**：管理buffer生命周期（QUEUED→DEQUEUED→DONE）③ **驱动层**：v4l2_device（主设备抽象）+v4l2_subdev（子设备抽象如Sensor/ISP）——**硬件无关的通用抽象层**。这是Linux内核设计的精髓——驱动开发者只需实现回调函数 ④ **硬件层**：Sensor（图像传感器·OV5640/IMX219）→ISP（图像信号处理·白平衡/去噪）→CSI（相机串行接口·MIPI CSI-2）→SoC（RK3588） |
| **▸ VB2三维抽象** | Queue（管理队列·控制buffer生命周期）/ Buffer（物理内存·一帧图像数据）/ Plane（子划分·YUV中Y平面和UV平面分开存储）——**三维设计的意义**：将"内存管理"和"数据格式"解耦 |
| **▸ Media Controller** | Entity（实体）+Pad（焊盘）+Link（链接）→用户态动态配置数据流向——**这是嵌入式视觉系统的"路由协议"**。**实战价值**：你可以在用户空间重新连接Sensor→ISP→DMA的pipeline而不修改驱动代码 |

| 颗粒度 | 细化内容 |
|--------|----------|
| **文件** | `./极致端侧本土化架构智能穿戴眼镜.md` |
| **▸ 三维预算·详细约束** | ① **算力预算**：NPU 0.5-2 TFLOPS→INT8量化后单次推理延迟10-30ms——实时性勉强达标（30fps=33ms/帧）② **内存预算**：模型量化后15-30MB——需与系统共享DDR·需要精细内存管理（CMA预留/ION分配）③ **功耗预算**：多级唤醒——VAD（Always-on·微瓦级）→关键词检测→NPU轻量推理（毫瓦级）→主模型推理（百毫瓦级）→执行——绝大多数时间停留在VAD级 |
| **▸ 架构演变** | 2026节点：TinyBERT（判别式·仅编码器）已受限→**MobileLLM 125M/350M**（生成式+多模态·能理解场景并生成自然语言反馈）更优。**演进逻辑**：从"识别这是什么"→"理解这个场景+生成合适的回应" |
| **▸ 多级唤醒链** | VAD（Always-on）→关键词检测（"Hey Glass"）→NPU轻量推理（确认用户意图·<10ms）→主模型推理（<200ms·生成回复）→TTS反馈——每级功耗逐级上升·**90%时间停留在前两级** |

## 🔷 5.1.3 PAN构想与架构 `[科技][PAN]`

| 颗粒度 | 细化内容 |
|--------|----------|
| **文件** | `./[科技][PAN]梳理我的PAN构想，软硬件实现逻辑，整体架构，技术与实施详细设计.md` |
| **▸ PAN定义** | Personal Area Network——概念已超越传统网络：**断网可用+隐私保护+低延迟的本地智能系统**。核心理念：不依赖任何云端的个人AI基础设施 |
| **▸ 五大核心特性** | ① 不依赖云端（断网可用·本地推理）② 隐私数据不出设备（所有数据本地处理·不传云端）③ 毫秒级响应（无网络往返延迟）④ 个性化模型持续本地微调（你的数据一直在本地·模型越来越懂你）⑤ 多设备协同（眼镜+手表+手机+PC=统一PAN网络） |
| **▸ 实施路径** | RK3588开发板（硬件底座）→ V4L2摄像头驱动（视频输入）→ NPU模型部署（AI推理）→ 多模态交互（语音+视觉+触控）→ **产品化**（智能眼镜/穿戴设备） |
| **▸ 泡沫辩证** | 对标1990s互联网泡沫——估值短期过热但**基础为真**（端侧AI是真实需求·不是炒作）。**关注技术迭代**（NPU架构演进/模型压缩算法）而非单纯扩产 |
