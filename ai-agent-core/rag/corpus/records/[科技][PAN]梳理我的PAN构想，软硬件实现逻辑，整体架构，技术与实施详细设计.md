#### 一、 核心逻辑：AI大基建的底层驱动
AI大基建（AI Infrastructure）本质上是算力集群的构建。如果说数据是“原油”，那么芯片、PCB、光模块和交换机就是“炼油厂”与“输油管”。

* **争议点：** 这种高额投入能否在应用端（C端、B端软件）获得足够的利润回报？即“投入产出比”的倒挂是否会引发泡沫破裂。

* **客观结论：** 当前处于“硬件先行”阶段。基建板块（芯片、光模块等）的业绩是真实的（实打实的订单），但**估值中包含了对未来应用端爆发的极高预期。**

### 二、 Python 代码实现

```python
import os
import requests
from google import genai
from langchain_text_splitters import RecursiveCharacterTextSplitter

# --- 配置区 ---
NOTION_TOKEN = "your_notion_integration_token"
DATABASE_ID = "your_database_id"
GEMINI_API_KEY = "your_gemini_api_key"

# 初始化 Gemini 客户端
genai_client = genai.Client(api_key=GEMINI_API_KEY)

class PANDataProcessor:
    def __init__(self):
        self.headers = {
            "Authorization": f"Bearer {NOTION_TOKEN}",
            "Notion-Version": "2026-03-11", # 使用最新的支持 Markdown 的版本
            "Content-Type": "application/json"
        }
        # 配置切片器：针对个人想法，建议块大小在 500-800 字符
        self.splitter = RecursiveCharacterTextSplitter(
            chunk_size=600,
            chunk_overlap=100,
            separators=["\n\n", "\n", "。", "！", "？"]
        )

    def fetch_notion_pages(self):
        """获取数据库中所有页面的 ID"""
        url = f"https://api.notion.com/v1/databases/{DATABASE_ID}/query"
        res = requests.post(url, headers=self.headers).json()
        return [page["id"] for page in res.get("results", [])]

    def get_page_markdown(self, page_id):
        """利用 2026 新接口直接获取页面的增强 Markdown 内容"""
        url = f"https://api.notion.com/v1/pages/{page_id}/markdown"
        res = requests.get(url, headers=self.headers).json()
        return res.get("markdown", "")

    def clean_and_split(self, text):
        """清洗 Markdown 特殊字符并进行语义切片"""
        # 简单清洗：可以根据需要剔除特定的 Markdown 语法
        # 这里直接利用 RecursiveCharacterTextSplitter 处理 Markdown 结构
        chunks = self.splitter.split_text(text)
        return chunks

    def get_embeddings(self, chunks):
        """调用 Gemini 批量生成向量"""
        # 使用最新的 multimodal embedding 模型，支持 8k token
        result = genai_client.models.embed_content(
            model="gemini-embedding-2",
            contents=chunks,
            config={"task_type": "RETRIEVAL_DOCUMENT"}
        )
        return result.embeddings

    def run(self):
        print("开始同步 Notion 数据...")
        page_ids = self.fetch_notion_pages()
        
        all_vector_data = []
        for pid in page_ids:
            raw_md = self.get_page_markdown(pid)
            chunks = self.clean_and_split(raw_md)
            embeddings = self.get_embeddings(chunks)
            
            # 整合元数据以便存储
            for i, emb in enumerate(embeddings):
                all_vector_data.append({
                    "id": f"{pid}_{i}",
                    "vector": emb.values,
                    "content": chunks[i],
                    "metadata": {"source": "notion", "page_id": pid}
                })
        
        print(f"处理完成，共生成 {len(all_vector_data)} 个知识片段。")
        return all_vector_data

# --- 执行 ---
if __name__ == "__main__":
    processor = PANDataProcessor()
    processed_data = processor.run()
    # 此处接入你已完成的存储逻辑，例如：vector_db.upsert(processed_data)
```

#### 2. 网络拓扑与物理边界的冲突（Static vs. Mobile）
* **现象：** 智能家居、办公电脑需要一个局域网内**“永远在线、IP固定”**的锚点来维持长连接（Keep-Alive）。
* **后果：** 手机是移动的。当你出门上班、或者进入地铁信号断开时，如果手机是 Hub，全屋的智能场景、汽车的联动将瞬间“群龙无首”坠入离线状态。Hub 必须像服务器一样，坚固地钉在物理空间的中心。

#### 3. 数据主权与隐私安全（Data Sovereignty）
* **现象：** 手机极易丢失、被盗，且手机操作系统（iOS/Android）掌控在苹果和 Google 手里，应用层权限经常被非法抓取。
