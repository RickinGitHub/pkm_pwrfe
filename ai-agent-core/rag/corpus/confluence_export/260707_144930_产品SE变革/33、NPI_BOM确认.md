# 33、NPI BOM确认

> pageId: 243464204 | 导出时间: 2026-07-07T14:52:06.548224

角色A更新时间
 
BOM工程师、产品开发经理BOM工程师、产品开发经理、软件项目经理具体内容对BOM信息中软件部分进行确认操作方法
1、软件leader 根据BOM订单信息评审出该BOM对应的PID，并在PDM上载软件ISN号（包含PID、版本分支信息）；
2、软件leader 创建该BOM对应的MID信息（clienttype、各种key【见附图】），并提交产品开发经理审核；
3、BOM工程师对该BOM流程做最终审核，在审核前发现软件信息有误或遗漏的情况，可以直接通知BOM工程师更改相关信息（不用出UECN更改）；
4、如软件leader 在BOM审核完成后发现有误，需变更BOM，则邮件通知软件项目经理出UECN变更流程；
5、BOM工程师对UECN变更流程做最终审核。
标准与原则PDM系统软件BOM3签字流程适用项目阶段
TR、PR，MP、IP
适用项目范围S，A，B，C，D
