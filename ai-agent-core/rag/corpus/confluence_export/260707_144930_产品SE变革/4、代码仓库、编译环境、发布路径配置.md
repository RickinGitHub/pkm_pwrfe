# 4、代码仓库、编译环境、发布路径配置

> pageId: 243464171 | 导出时间: 2026-07-07T14:51:32.772387

角色A更新时间
 
沟通对象配置工程师，集成工程师具体内容编译环境准备，代码管理及项目配置操作方法1.软件Leader根据项目需求，评估是否新增源码库、编译工具链、Jenkins构建、发布空间等，如需要，则按照软件技术中心配置管理规范申请；
2.软件Leader将项目源码上传到源码库，并确保代码clone、commit、push以及编译等正常通过，如不能通过则与配置工程师沟通确认；
3.软件Leader根据项目需求，配置项目人员的产品源码库权限，一般给模块组长赋予Maintainer权限，然后由模块组长给其他团队成员赋予Developer权限；如果没有账号则需要根据Gitlab权限申请格式填写，并发邮件给配置工程师开通账户和相关权限；
4.软件Leader定期梳理相关人员的权限，避免无评审提交；
5.软件Leader与集成工程师沟通并由集成工程师配置Jenkins构建、发布空间等相关事宜。标准与原则
