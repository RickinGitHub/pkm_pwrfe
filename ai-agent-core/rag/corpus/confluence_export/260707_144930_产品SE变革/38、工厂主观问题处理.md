# 38、工厂主观问题处理

> pageId: 243464214 | 导出时间: 2026-07-07T14:52:09.540317

角色A更新时间
 
沟通对象SQA测试工程师，QPL（NPI质量工程师） ，AOE工程师，FQA测试工程师，模块开发owner具体内容回复PR主观报告中的软件相关问题点处理意见、原因分析及整改计划操作方法
1、FQA 测试完成后输出工厂主观报告，主观报告中需要包含软件版本、机型、和ProjectID，AOE电子分析工程师与FQA、QPL对机梳理出需要软件确认的问题，并发给对应的软件Leader（一版为标黄或标红的问题）；

2、对于报告中描述不清、无法复现的问题需安排与AOE、QPL、FQA现场对机确认；

3、软件leader 同步主观报告到SQA，由SQA 确认软件问题的概率、复现手法，并上mantis或Jira；

4、查收SQA的核对结果，将可复现的问题分派给各模块开发owner，并要求他们及时提供分析结论或整改计划。针对不影响用户使用问题可建议质量接受，如发现严重问题，需提前与QPL沟通问题风险；

5、将SQA的核对结果及可复现问题的分析结论或整改计划，写入PR主观报告中，回复给AOE工程师；

6、针对各方认为无法接受释放的问题，需要组织产品开发经理、QPL、SQA测试Leader、产品软件部门长及SQA部门长进行评审；

7、针对承诺整改的问题，应该按计划输出改善软件，并及时安排SQA测试工程师进行回归测试。
标准与原则
适用项目阶段PR、IP、MP适用项目范围S、A、B、C、D交付件
