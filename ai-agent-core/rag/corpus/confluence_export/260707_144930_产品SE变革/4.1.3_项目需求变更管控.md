# 4.1.3 项目需求变更管控

> pageId: 246759398 | 导出时间: 2026-07-07T14:50:35.680495

| **标题** | 需求变更管控 |
| --- | --- |
| 项目阶段 | 全生命周期 |
| 角色 | R |
| 输入 | 《项目需求变更申请》 |
| 沟通对象 | 需求分析师(BA)、开发工程师、软件项目经理 |
| 具体内容 | 需求分析师根据项目变更需求，组织需求变更提出方、模块owner、产品SE、SQA进行需求变更评审 |
| 操作方法 | 1、需求分析师统一接收项目变更需求；  2、需求分析师对需求变更进行解读，明确需求功能变更项，组织软件相关团队对需求进行评估。  3、模块SE根据需求进行技术方案的验证，输出整体技术方案、变更风险、工作量、时间计划  5、SQA输出需求变更测试策略  5、需求分析师整理输出《需求变更评估报告》，组织模块SE、产品SE、质量SE、SQA进行评审  6、需求分析师依据《需求变更评估报告》，和整机产品开发经理达成一致，由软件部部长签批。  7、软件项目经理根据需求变更计划，更新项目开发计划，并跟进闭环  8、需求分析师输出《软件需求变更记录表》。 |
| 标准与原则 |  |
| 指导文件 | 《项目需求变更申请》模板、《需求变更评估报告》模板、《软件评估报告》模板 |
| 适用项目范围 | S,A,B项目 |
| 交付件 | 《需求变更评估报告》 |
