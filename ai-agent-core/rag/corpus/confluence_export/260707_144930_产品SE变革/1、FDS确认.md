# 1、FDS确认

> pageId: 243464167 | 导出时间: 2026-07-07T14:51:31.277813

角色A更新时间
 
沟通对象产品开发经理、软件开发经理、软件项目经理、各专业负责人具体内容FDS各项功能确认，识别区域、版本、新功能差异等。操作方法
1、参与确认FDS各项功能；
2、识别功能差异（包括区域差异等）、核心应用、关键功能，为后期项目计划及测试计划做准备；

3、在全功能软件输出前，软件Leader需要根据FDS能逐个确认功能是否完成开发，避免遗漏。
标准与原则
适用项目阶段
concept、kick off
适用项目范围S、A、B、C、D交付件
