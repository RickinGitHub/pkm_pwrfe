# 1.项目风险会

> pageId: 243464050 | 导出时间: 2026-07-07T14:52:59.086964

| 标题 | 项目风险会 |
| --- | --- |
| 状态 | 待评审 |
| 角色 | A |
| 更新时间 |   |
| 沟通对象 | 软件项目经理、产品SE、VPM、各软件模块Owner |
| 具体内容 | 召开项目风险会，评估当前项目风险，输出相应的应对措施和解决方案 |
| 操作方法 | 1、根据项目进度节点，与软件项目经理、产品SE、VPM制定周期性项目风险会召开时间，项目量产前两个月开始周期性召开项目风险会； 2、软件项目经理、产品SE 与VPM确认当前项目问题和测试问题，与各软件模块Owner确认重点必解问题解决情况，并要求输出问题解决计划； 3、软件项目经理、产品SE、VPM 按时召开项目风险会，梳理当前项目项目问题和测试问题，并评估当前项目风险点； 4、软件项目经理整理会议记录，输出项目风险表，邮件给各参会人员； 5、软件项目经理跟进项目风险点，推动各负责人按计划完成风险点的梳理解决。 |
| 标准与原则 | 按软件中心项目管理模板输出 |
| 适用项目阶段 | SR4 SR5 |
| 适用项目范围 | S、A、B、C |
