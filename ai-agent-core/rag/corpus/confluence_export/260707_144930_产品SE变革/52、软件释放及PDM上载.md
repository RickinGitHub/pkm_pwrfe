# 52、软件释放及PDM上载

> pageId: 243464236 | 导出时间: 2026-07-07T14:52:31.100685

角色A更新时间
 
沟通对象AOE，QPL具体内容软件释放单的释放及修改操作方法
1、签署软件释放单

    a、软件主观报告PASS流程完成后，AOE在PDM系统发起软件释放单签署流程，并转交给软件leader

    b、签署流程流转到软件leader时，会有邮件通知软件leader签署，此时登陆PDM系统，在我的工作→我处理的工作流→当前流程中找到对应软件释放单申请流程，双击打开流程，确认试产批次号，PID及BOM信息无误后，点击提交，流程会自动转交到QPL进行签署，待QPL及后面的所有签署流程完成后，软件释放单就正式生效。

2、修改软件释放单

    a、软件释放单生效后，会存在需要增加或删减BOM的情况，此时需要修改软件释放单，登陆PDM，在部品管理->软件管理→软件释放单修改→试产批次中通过试产批次号获取对应软件释放单的具体信息

    b、释放单信息界面中的适用BOM清单区域会列出当前释放单支持的BOM列表，对于删除BOM的情况，在此区域中找到对应的BOM，点击删除行按键进行删除操作，对于增加BOM的情况，在此区域中点击新增行按键进行查找界面

    c、在查找界面的TCLNum输入框输入PID查找到PID对应的信息，选中并确认后，会在释放单的BOM清单区域中增加此PID对应的所有BOM，对于不需要的BOM，通过删除行按键进行删除，只保留需要增加的BOM即可。

    d、在适用BOM清单区域增加或删除BOM后，还需要注意软件版本及其对应CPP芯片清单区域的软件虚拟物编信息是否正确，如果不正确需要修正

    e、上面的信息确认完后，点击保存及提交按键进入提交界面，并在QE处理人框中选择QPL作为下一步的流程签署人，选择完成后提交即可。
标准与原则删除和增加BOM时，只删除和增加指定的BOM，不能操作其它BOM适用项目阶段
