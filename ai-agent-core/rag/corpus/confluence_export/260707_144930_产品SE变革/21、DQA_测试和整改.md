# 21、DQA 测试和整改

> pageId: 243464190 | 导出时间: 2026-07-07T14:51:48.795122

角色A更新时间
 
沟通对象硬件leader，软件leader, 外协，模块负责人，产品开发经理，AOE工程师具体内容解决电测中的软件问题（时序控制，电性能，电声，认证，能效，ESD）操作方法
1.与硬件leader沟通确定各测试的节点时间，以及每一个测试阶段的具体要求，制定对应的开发计划并按时输出测试软件；
2.针对测试要求，安排项目组成员做好软件的提前整改；
3.如果需要底层soc芯片厂商性能改善，硬件Leader推动外协协助整改；（软件项目管理跟进）
4.软件leader安排项目组输出整改后的软件给到硬件；
5.硬件使用整改后的软件测试直至通过；（软件项目管理跟进）
标准与原则硬件性能测试问题列为高优先级。适用项目阶段
LR、TR、PR
适用项目范围S、A、B、C、D交付件
