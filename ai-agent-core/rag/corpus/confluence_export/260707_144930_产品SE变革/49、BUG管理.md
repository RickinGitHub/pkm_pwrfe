# 49、BUG管理

> pageId: 243464232 | 导出时间: 2026-07-07T14:52:26.440411

角色A更新时间
 
沟通对象产品开发经理、QPL、测试leader、测试工程师、软件项目经理、开发工程师具体内容Review mantis  BUG,  跟进bug问题流转、解决，满足项目进度要求 操作方法
1、对测试工程师提交的mantis bug进行跟进，明确Due Date，测试工程师对问题分配不清楚时由软件leader统一分配。
2、与产品开发经理、测试leader、QPL针对问题严重性是否打必解标签
3、组织晨会，与软件项目经理、SOC接口人Review 问题解决进展。
4、组织产品开发经理、QPL、测试leader、对SCCB、moreinfo问题评审

5、项目量产前1-2个月，由部门长(含各专业部门长)参与每日遗留严重问题review，提高问题解决效率
6、在项目推进过程中，重复步骤1、2、3、4、5直至问题关闭或者有明确结论，不能影响项目量产。
标准与原则1、问题流转到相应工程师，相应工程师必须在三天内做出问题分析。
2、BUG流转需有分析过程注释适用项目阶段
kick off
适用项目范围S，A，B，C、D交付件
