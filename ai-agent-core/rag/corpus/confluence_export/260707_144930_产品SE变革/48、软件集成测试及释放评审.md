# 48、软件集成测试及释放评审

> pageId: 243464231 | 导出时间: 2026-07-07T14:52:25.939788

+

角色A更新时间
 
沟通对象SQA测试Leader，SQA部长，产品部长，产品开发经理，NPS部长具体内容1、申请版本集成并通知SQA测试Leader安排测试 ;
2、组织SQA测试Leader，SQA部长，产品开发经理及产品部长对问题进行评审。操作方法
1、软件LEADER与SQA测试LEADER沟通版本测试计划与测试资源，并在海外软件版本发布大盘表填写版本发布计划；
2、软件LEADER发邮件给集成人员申请软件集成；

3、集成人员集成好软件后，要先做冒烟测试，冒烟PASS后告知软件Leader填写测试申请表；
4、软件LEADER填写测试申请表，此表包含：

      ReleaseNote（遵循软件集成ReleaseNote规范）:

       Mantis的Release问题，产品、中间件及应用等模块的修改记录。

      测试建议:

       测试的重点模块以及区域优先级等信息。

     待版本集成后通知SQA测试LEADER根据测试申请表的修改情况及测试建议启动版本测试。

5、SQA发出测试报告后，软件LEADER初步过滤报告是否有影响版本释放的问题，如果有严重问题，则需要解决问题并重新集成发布版本。

6、通知SQA测试LEADER筛选影响版本释放的问题，软件LEADER评估问题的风险，如果风险可控，则编写版本仲裁单，将各个问题的原因及影响范围写清楚并签名后发送给SQA测试LEADER。

7、测试LEADER根据软件LEADER的分析结论，筛选出需要上升评审的问题。
8、软件LEADER组织测试LEADER，SQA部长，产品部长对问题进行评审，如果评审结果可以释放，则由SQA部长及产品部长对仲裁单进行签名并释放软件；如果评审结论有争议，则需上升到NPS部长进行仲裁；

9、软件测试或评审通过后，软件LEADER根据需求发邮件给集成人员制作预抄写申请；
10、软件LEADER填写预抄写测试申请表；
11、软件测试或评审通过后，由SQA测试Leader将软件(升级包及预抄写文件)放置到软件发布路径，并对工厂发布。

12、软件如果是法国TQC释放，除了SQA测试通过外，TQC测试也要通过，而且还要提供release note 表格 、ProjectID_Release表格、FOTA升级包、SQA测试结果。共4项给到法国TQC 。
