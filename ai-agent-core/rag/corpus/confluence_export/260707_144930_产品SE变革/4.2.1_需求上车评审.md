# 4.2.1  需求上车评审

> pageId: 245927611 | 导出时间: 2026-07-07T14:50:36.183428

| 标题 | 需求上车评审 |
| --- | --- |
| 状态 | 待评审 |
| 角色 | A |
| 更新时间 |    |
| 沟通对象 | 需求分析师（BA）、软件产品SE、软件项目经理、开发工程师、测试工程师、质量SE |
| 输入 | 《需求上车评审表》 |
| 具体内容 | 1.参与新需求上车评审  2. 评估需求是否存在风险  3. 评估是否存在Block/重点问题，对整机软件影响 |
| 操作方法 | 1、检查需求评审上车要素表是否齐全（需求矩阵、测试用例、自测报告、SQA测试报告、性能、稳定性、安全、集成说明）  2、确认需求矩阵设计是否符合用户操作、使用习惯  3、确认测试用例是否覆盖完全  4、检查自测、SQA测试报告是否通过，对未关闭问题进行评审，确认是否有未解决问题，如有不允许上车；  5、性能、稳定性如有劣化不允许上车  6、涉及安全问题的需整改后才允许上车  7、根据集成说明进行Feature功能配置，整机系统相关配置，安排CIE集成导入； |
| 标准与原则 | 新需求功能符合用户体验，交付件齐全，方案测试通过，性能、稳定性、安全符合设计要求， |

| 交付件 | 评审结论 |
