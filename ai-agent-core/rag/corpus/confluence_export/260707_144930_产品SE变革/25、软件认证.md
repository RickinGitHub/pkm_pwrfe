# 25、软件认证

> pageId: 243464194 | 导出时间: 2026-07-07T14:51:50.768492

角色A更新时间
 
沟通对象认证经理，认证技术owner，认证测试leader，软件项目经理，整机产品开发经理，软件leader具体内容组织评估项目中重大认证的工作量，制定认证总体进度计划操作方法

1. 项目立项后与产品开发经理确认项目中与软件相关的认证内容项和认证版本；
2. 各沟通对象，依据认证内容和认证节点讨论并制定认证进度计划以及梳理和评估认证风险；
3. 认证进度计划需由项目团队成员共同确认认可的；
4. 软件leader 需依据认证进度计划按时输出各阶段的认证软件，并协调人力、设备等资源满足测试要求，认证技术owner负责跟进问题解决，由认证经理和软件项目经理协助跟进；
5. 如不能按计划输出软件，软件leader 需提前通知认证测试leader并依据情况上报风险及重新调整认证计划；
6. 对于外送机器的认证，认证经理需要汇总外送机器的硬件、软件状态以满足后续升级、调试等要求；
7. 认证经理和软件leader把控进度风险，认证技术owner梳理认证中遇到的问题及提供技术支持，承担交付认证的责任；
8. 认证测试leader 需安排人力测试并按时输出测试结果，测试过程中发现重大或阻碍认证进行的问题需及时告知认证技术经理和软件leader；
9. 认证测试leader，认证技术owner，认证经理共同评审软件是否可以外送；
