# 34、客户样机问题反馈

> pageId: 243464208 | 导出时间: 2026-07-07T14:52:07.568634

角色A更新时间
 
沟通对象沟通对象包含但不限于客户、整机产品开发经理、软件开发经理、软件项目经理、整机项目管理具体内容确认场测问题识别出软件问题，组织判定场测问题的等级，按照优先级输出整改计划并跟踪解决操作方法
1、确认客户样机状态（硬件方面包括：机型，机芯，配件；软件方面包括：软件版本，PID，外设软件版本）

2、如客户样机硬件状态 跟 工厂生产机器不一样，需跟整机项目管理、整机开发经理一起讨论是否需要邮寄最新硬件状态的机器，如软件评估有必要可要求邮寄

3、识别客户提报问题，筛选出软件问题并上载mantis 或Jira 统一管理，组织输出软件问题的整改计划并跟踪解决，整改计划中的解决时间需结合量产或者订单要求

4、对于客户提报问题中涉及新需求，新规范的问题，需上升到整机产品开发经理和软件开发经理

     a、新需求走软件新需求导入流程

     b、新规范需软件leader、软件开发经理一起组织相关专业评估输出整改时间
标准与原则识别客户需求的优先级，尽可能优先处理此类问题，避免引发客户不满适用项目阶段
TR、PR、IP
适用项目范围S、A、B、C、D交付件
