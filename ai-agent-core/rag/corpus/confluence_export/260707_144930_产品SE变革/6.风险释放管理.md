# 6.风险释放管理

> pageId: 243464056 | 导出时间: 2026-07-07T14:53:11.084917

| 标题 | 风险释放管理 |
| --- | --- |
| 状态 | 待评审 |
| 角色 | A |
| 更新时间 |   |
| 沟通对象 | 产品开发经理、SQA部长、产品部长、VPM、QPL、工厂计划 |
| 具体内容 | 对SQA提出问题进行风险评估，并释放；在生产计划紧急，但软件试产未释放的情况下，进行风投评估。 |
| 操作方法 | 一、软件风险释放（SQA测试阶段）：  1、对jira上有争议的bug，产品SE、VPM需要首先进行初评审； 2、对于有风险的问题，组织SQA部长、产品部长、产品开发经理、QPL进行评审，若评审仍存在分歧，则需要NPS部长仲裁；  3、对于评估可以风险释放的问题，列出整改时间，及网络升级计划，由软件项目经理跟进；  二、在生产计划紧急，但软件试产未释放的情况下，进行风投评估（风投）：  1、工厂计划人员发起申请，列出风投原因、风险规避措施、风投责任承担部门；  2、产品SE评估软件风投风险，并与产品组组长、产品部长沟通评估决策； |
