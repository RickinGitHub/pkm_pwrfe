# 4.2.3 需求闭环

> pageId: 245928104 | 导出时间: 2026-07-07T14:50:38.187722

| **标题** | 需求闭环 |
| --- | --- |
| **状态** | 待评审 |
| **阶段** | SR4-2 |
| **角色** | A |
| **输入** | 《项目需求表》 |
| **沟通对象** | 需求分析师(BA)、VPM、产品SE、软件项目经理 |
| **具体内容** | 对项目的需求进行验收 |
| **操作方法** | 1、VPM组织，和产品SE一起对新需求制定测试策略。  2、BA 将新需求更新到FDS。  3、由软件项目经理发起,需求分析师在SR4-2对需求进行验收。  4、产品SE确认项目需求配置。  5、产品SE根据需求测试情况评估是否满足释放标准。 |
| **标准与原则** |  |
| **指导文件** | 《需求释放标准》 |
| **适用范围** | S,A,B,C,D类项目 |
| **交付件** | 《功能导入表》 |
