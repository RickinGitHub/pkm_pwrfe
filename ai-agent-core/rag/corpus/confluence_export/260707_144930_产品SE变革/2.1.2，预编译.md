> pageId: 245932272 | 导出时间: 2026-07-07T14:53:20.044766

| **标题** | **预编译** |
| --- | --- |
| 项目阶段 | 项目各阶段 |
| 输入 | 预编译需求 |
| 沟通对象 | 工程效能工程师，CIE工程师 |
| 具体内容 | 编译环境准备，代码管理及项目配置 |
| 操作方法 | 1.产品SE根据项目需求，评估是否需要搭建预编译环境；2.产品SE与工程效能工程师沟通并由集成工程师配置Jenkins构建，确保项目manifest和分支及编译命令正确；  3.预编译环境搭建完成以后，触发方式：使用patch_delivery提交工具自动触发预编译(会自动填写uploadok!!) 或者在代码提交的gerrit上填写**uploadok 或者uploadok!! **进行预编译；4.项目预编译搭建好以后，如果出现排队时间长、单patch编译时间长、预编译报错等情况，产品SE及时反馈给工程效能工程师改善； |
| 标准与原则 | <1>所有合入到主干分支的提交都要经过预编译成功，保证分支的质量 |
