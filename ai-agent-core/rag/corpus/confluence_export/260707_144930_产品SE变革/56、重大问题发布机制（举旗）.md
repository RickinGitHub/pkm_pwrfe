# 56、重大问题发布机制（举旗）

> pageId: 243464242 | 导出时间: 2026-07-07T14:52:39.050893

角色A更新时间
 
沟通对象各项目软件Leader，部门负责人具体内容重大问题共享操作方法
1、基础机芯软件Leader对发现的重大问题进行邮件发布通告，通知其他派生项目leader同步导入，并邮件抄送部门负责人；
2、派生项目软件Leader对发现的重大问题，需与基础机芯软件Leader进行评审，如确认需导入全机芯平台，则由基础机芯软件Leader邮件发布通告，通知其他派生项目leader同步导入，并邮件抄送部门负责人；
3、不分机芯平台的问题，软件Leader触发升级到部门负责人，部门负责人组织制定各个机芯平台问题的检查措施及解决方案，避免其他机芯或区域出现同类问题；
4、软件Leader确认是否有此重大问题的测试用例，如没有，通知SQA添加测试用例，并参与测试用例评审。
标准与原则
适用项目阶段
LR，TR，PR，IP，MP
适用项目范围S、A、B、C、D交付件
