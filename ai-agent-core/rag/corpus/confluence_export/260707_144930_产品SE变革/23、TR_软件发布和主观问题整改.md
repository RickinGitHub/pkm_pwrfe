# 23、TR 软件发布和主观问题整改

> pageId: 243464192 | 导出时间: 2026-07-07T14:51:49.773201

角色A更新时间
 
沟通对象AOE工程师、硬件工程师、产品开发经理、整机项目管理、PQ 工程师等、模块owner具体内容TR软件发布及TR试产问题整改，保证TR评审顺利通过，进入PR阶段操作方法
1、项目立项后，项目管理根据项目要求拉出TR,PR,IP的时间节点

2、软件leader在TR时间节点前，发布TR 软件并安排测试(发布软件流程与IP 量产软件一致）。

3、测试通过后安排集成人员制作TR试产的预抄写软件验证通过后提供工厂TR试产软件路径

4、TR试产反馈的问题，识别软件问题，确认是问题的推动模块开发工程师解决，非问题与AOE工程师、整机项目管理、产品开发经理沟通澄清

5、针对无法准确定位为软件类问题， 需反馈给整机开发经理和AOE，让他们组织各个专业一起会诊；

6、TR试产中无法解决或者无法按时解决的问题需要与AOE、QPL、产品开发经理评审并达成一致
标准与原则
适用项目阶段
TR
适用项目范围S、A、B交付件
