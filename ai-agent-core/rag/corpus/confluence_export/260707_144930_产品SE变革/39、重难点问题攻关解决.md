# 39、重难点问题攻关解决

> pageId: 243464215 | 导出时间: 2026-07-07T14:52:10.054577

角色A更新时间
 
沟通对象模块开发Owner、软件项目经理、QPL、产品开发经理、SQA测试Leader具体内容评估重难点问题的解决计划，量产前必解问题须集中优势资源进行攻关解决操作方法
1、软件Leader梳理mantis（Jira）上的遗留bug，组织软件项目经理、QPL、产品开发经理、SQA测试Leader等评估必解问题，并设置必解问题的deadline，评估问题解决计划时，应该综合考虑项目的量产计划，充分评估并圈定影响量产的问题，不阻塞量产的低风险问题建议后续迭代导入，首先保证量产软件的按时交付；

2、针对概率性问题、压测问题等重难点问题，需要制定详细的测试、分析、解决计划，特别是量产前必解问题需要每天上晨会追踪进度，连续2天无进展的问题应该及时组织专家（系统工程师、模块负责人、开发组长及其他专家）讨论并制定下一步分析方向、实验方案等；

3、针对无法按时解决的问题，应该提前暴露风险，并与QPL、产品开发经理、开发组长、软件部门长、SQA部门长充分评估问题风险，并决策软件是否进行风险释放。
