# 28、新需求导入

> pageId: 243464197 | 导出时间: 2026-07-07T14:51:52.267637

角色A更新时间
 
沟通对象软件开发经理，软件项目经理，各专业负责人，产品开发经理，软件认证经理，软件测试Leader具体内容根据新需求（含新功能、新应用、各类派生项目）开发及导入流程，软件Leader参与新需求的评估，并承担与整机系统、工厂生产部分的开发及方案制定工作，阶段性发布新功能测试版本安排测试，在测试通过后导入量产软件操作方法
1、软件开发经理接收新需求，组织各软件专业（含软件Leader）进行需求评估，并输出《新需求评估报告》，含技术方案、进度、人力及风险等内容；
2、各软件专业输出开发WBS；
3、各软件专业输出《需求矩阵》、《自测用例》或《测试用例》；
4、各软件专业在整机主干上进行开发，软件项目经理进行开发过程管控；
5、若导入的需求是新应用，则由应用owner自行提测，若导入的需求需要发布系统版本进行测试的情况，则由软件Leader根据开发进度阶段性发布软件，进行新功能提测；
6、SQA测试并输出《测试报告》，若测试通过，则由软件Leader安排各开发人员整理《新功能产品集成表》，软件Leader根据整机情况并参照《新功能产品集成表》，安排新功能合入量产分支；
标准与原则新需求导入流程，请参考《海外新功能需求管控流程》适用项目阶段
PR、IP、MP
适用项目范围S、A、B、C、D交付件《新需求评估报告》、《需求矩阵》、《自测用例》或《测试用例》、《新功能产品集成表》备注
 添加《新功能产品集成表》，提高产品集成效率
