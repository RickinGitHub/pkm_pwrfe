# 4.2 SCCB评审

> pageId: 245933794 | 导出时间: 2026-07-07T14:53:26.553732

| **标题** | SCCB评审 |
| --- | --- |
| **状态** |  |
| **角色** | A |
| **更新时间** |   |
| **沟通对象** | 质量SE、SPM、产品SE、VPL、模块Owner |
| **具体内容** | 针对项目SCCB问题评审 |
| **操作方法** | - **SCCB评审**        SPM召集产品SE、质量SE、VPL、模块Owner针对SCCB状态问题进行梳理：        1. 多版本未复现问题按流程梳理，评审符合标准直接关闭；        2. 模块Owner与模块SE分析为正常逻辑或非TV类问题，评审通过直接关闭；        3. 模块Owner评估影响较小或者可接受问题，经产品SE、质量SE、VPL评审通过的问题可直接关闭；            评审有疑义问题，采用如下策略：                      a. 需要进一步澄清或验证事项，可留澄清事项或者重新转assigned；             b.针对模块owner认为设计或架构等因素走SCCB问题，VPL如不认可产生争议，可由VPL上升TSE进行评审；             c.上升TSE后继续存在争议问题，可由模块Owner上升SEG决策（）         4.评估需转需求问题，做转需求处理        产品SE负责对SCCB中填写的技术相关内容进行鉴别，保证SCCB解释内容合理和有效性，协助参评各方真实理解技术原理及SCCB原因        评审结论和遗留事项需要在jira单上进行备注记录 |
