# 3.整体技术方案的设计

> pageId: 243464048 | 导出时间: 2026-07-07T14:53:17.570217

| 标题 | 整体技术方案的设计 |
| --- | --- |
| 状态 | 评审中 |
| 角色 | A |
| 更新时间 |   |
| 沟通对象 | 整机开发代表、软件经理、软件BA、软件项目经理、软件产品SE、软件认证经理、各模块开发Owner |
| 具体内容 | 从整机的维度输出整机软件的整体技术设计方案 |
| 操作方法 | 1. 撰写《XXX软件项目整体方案设计》 报告之前，需要了解项目的整体需求和规格，包括但是不限于：项目背景、项目目标、产品定位、新品规划的核心卖点、本项目所使用的IC 及其他器件的规格及对标情况、需求规格说明书&XX 产品FDS、各模块功能开发计划和风险点、硬件(整机，包括主板、屏、外设等，了解硬件各外围方案的具体细节，包括型号，控制方式，硬件特殊需求等规格 2. 产品SE 需要熟练掌握本项目SOC方案的规格和架构（如CPU能力、GPU能力、APU算力、内存架构方案等）及跟上一代产品的对标差异，评估SOC方案是否能满足关键功能需求和技术方案的需求，如需进一步评估的项，需要制定评估计划 3. 《XXX软件项目整体方案设计》报告中需要呈现出本项目跟上一代产品（项目）差异的需求/功能的设计方案、核心功能的设计方案，以及实现这些功能/需求 所使用的方法/思路 4. 撰写《XXX软件项目整体方案设计》报告时，报告中需要包含所呈现的技术方案的核心抓手、核心抓手目标及应对策略，风险点以及风险应对措施 |
| 标准与原则 | 按照中心设计方案评审规范进行评审，包含但不限于如下： 1、参与评审人员要求至少有主任级及以上的工程师/系统工程师； 2、评审结论要包含责任人的具体工作和时间节点，软件项目经理帮忙跟进； 3、功能比较复杂的可组织与产品开发经理评估是否分阶段实现和交付； 4、设计方案要考虑兼容性、异常处理、性能和用户体验等； |
| 适用项目阶段 | Design |
| 适用项目范围 | S、A、B |
| 交付件 | 按解决方案中心的要去输出《XXX软件项目整体方案设计》、方案评审会议纪要 |
