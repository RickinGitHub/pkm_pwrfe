# 36、SCCB 问题 Review

> pageId: 243464211 | 导出时间: 2026-07-07T14:52:08.528314

角色A更新时间
 
沟通对象测试Leader，QPL，整机产品开发经理，软件leader具体内容
1、在项目开发或派生迭代阶段 SCCB 问题数量超过50，由软件项目经理组织问题review，及时清理 ；

2、评审问题前，软件leader 需要清楚所评审的bug 状态，包括bug 的出现的具体步骤、概率、可恢复性、以及功能设计流程等；

3、对每个S/A级问题， 评估影响程度， 对于影响比较轻微的， 降级为B级， Note标注；

4、对每个B级问题， 评估影响程度， 对于影响比较严重的， 升级为A级， Note标注；

5、对于SCCB评审结果是因为测试用例不准确的，由测试leader组织测试用例评审，修改测试用例，关闭问题；

6、对于SCCB评审结果是不能接受的问题，备注原因后问题状态改为Assign，并给出解决时间；

7、对于SCCB评审结果为可接受的问题，测试Leader 关闭此问题；
操作方法

标准与原则bug 不能影响工厂的生产、不能引起重大的用户投诉适用项目阶段
TR、PR、IP、MP
适用项目范围S、A、B、C、D交付件
