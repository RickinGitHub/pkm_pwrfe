# 2.3.1 分支申请

> pageId: 253024597 | 导出时间: 2026-07-07T14:53:24.040902

| 标题 | **分支申请** |
| --- | --- |
| 状态 | 已评审 |
| 角色 | A |
| 更新时间 |   |
| 沟通对象 | 运维工程师、CIE工程师、各组件接口人 |
| 具体内容 | 在单独的JIRA空间新建分支申请流程，核对分支拉取checklist保证拉取的整机、组件、闭源仓正确性。 |
| 操作方法 | 1、在JIRA空间（[泛智屏三分支管理](https://jira.tclking.com/projects/CMBR/summary)）提出拉分支需求：        a. 提出量产分支、主干分支、认证分支、开发分支的拉取需求和使用说明，是用于量产/测试/认证/开发自用       b. 提供基准分支路径        c. 核对各组件和闭源仓的信息  2、产品部门长和工程效能部门长根据分支管理规范，确认分支申请需求的合理性。  3、运维工程师根据jira单创建业务代码新分支，整理输出新分支代码下载路径。  4、 CIE工程师确认新分支的编译和冒烟结果，发布分支信息。  5、 产品SE向项目组发布新拉分支信息，包括：       a. 新分支集成冒烟结果      b.新分支代码下载路径      c. 修改的组件集成路径  6、产品SE需要和SOC对齐新拉分支后，SOC闭源代码需要同步拉新的分支。 |
| 标准与原则 | - 原则上标准的三分支（量产分支、主干分支、开发分支）由产品SE提出拉分支需求； - 开发分支是由主干拉出，在主干创建时，会从主干同步拉出开发分支； - 新项目完成预研开发，摸底测试通过经过评审，可以拉出主干分支。 -  量产分支从主干拉出，当项目稳定性达到一定标准时，SR5过点后可以从主干拉出量产分支，以管控代码的合入；  - 不同机芯共主干，同一机芯共版本； |
| 适用项目阶段 | 所有 |
| 适用项目范围 | S、A、B、C、D |
