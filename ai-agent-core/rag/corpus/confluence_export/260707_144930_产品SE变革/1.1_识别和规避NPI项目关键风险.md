# 1.1 识别和规避NPI项目关键风险

> pageId: 243464049 | 导出时间: 2026-07-07T14:50:21.940960

| **标题** | 识别和规避NPI项目关键风险 |
| --- | --- |
| 项目阶段 | SR3，SR4，SR5 |
| 输入 | 各个节点时间里程碑，需求列表，问题列表，需求设计方案 |
| 沟通对象 | VPM，开发owner，SOC，PM，BA，FDE，SQA，模块owner，SEG |
| 具体内容 | 在NPI项目中识别各个阶段风险以及制定相应规避措施。风险包括进度风险，技术风险，质量安全风险。 |
| 操作方法 | ### 1.制定出风险识别途径，怎么去快速识别到风险。eg:建立项目例会制度  [1.项目风险会](https://confluence.tclking.com/pages/viewpage.action?pageId=243464050)  ### 2.在SR3阶段主要是对软件整体设计方案的风险识别。包括器件选型是否能满足项目需求，重大需求上车是否来带性能&稳定性方面风险以及设计方案方面是否存在合规，安全风险等，项目计划定制是否过于激进达成上是否存在风险。  [2.需求导入风险](https://confluence.tclking.com/pages/viewpage.action?pageId=243464051)  [12.签BOM风险](https://confluence.tclking.com/pages/viewpage.action?pageId=246760843)  ### 3.到达SR4阶段主要识别到以下几个风险：  ####      3.1 需求遗漏风险。需要进行需求点检，确认需求是否100%上车，BA需要输出点检表（产品SE跟BA一同确认点检表内容是否全部已导入）。  ####      3.2 DI值风险。SR4过点对DI值，有明确要求，需要收敛到300以内。  ####      3.3 工厂适应性是否达标风险。新机芯Leading项目，需要和工厂IPL确认工厂生产适应性问题已全部解决，此过程需要扫清block工厂生产的问题。在此阶段，需要VPM同步提测DFM测试。  ####      3.4 性能，稳定性风险。此过程需要保证稳定性，性能都达标。切记一定要在需求全部上车后，提测性能，稳定性的专项测试。  ####     3.5 【E-Sticker资源】和【内置视频】准确性检查。此项需要找PDT确认清楚，需要确认内容如下：1.esticker 资源是否正确，是否得到前端认可。2.在esticker资源网站上是否有注册该机型类型信息以及对应资源适配。3. esticker xml是否有导入到项目中。  4.内置适配，esticker导入后需要安排压测。      3.6 场测风险。特别对于给google 进行DF测试的机器，一定要对base版本，机型，机器userDI进行重点登记和检查。后续推送ota升级需要单机部署，因此此类信息十分总要。 [5.PQ导入风险](https://confluence.tclking.com/pages/viewpage.action?pageId=243464054)  [6.风险释放管理](https://confluence.tclking.com/pages/viewpage.action?pageId=243464056) [9.场测风险](https://confluence.tclking.com/pages/viewpage.action?pageId=246760811)  ### 4.进入SR5阶段，软件已经比较成熟，接近量产状态。因此这过程也有几个风险点需要注意.  ####      4.1 量产分支质量看护，防止劣化风险。对于需要合入量产分支的问题精准把控其风险，防止带来稳定性，性能的劣化或者引入其他严重问题。对比不block释放的问题不带入。  ####     4.2确认PQ，电声参数是否有遗漏风险。特别是对于多款机型一定要找PQ，电声owner确认无误。  ####     4.3 DI值风险，此过程需要继续考核DI值中国区总DI值小于70，海外总DI值小于150；此过程需要对于block问题积极组织攻关，每日早晚会跟进，拉相关领域owner制定出行动项。快速收敛，达到释放要求。  ####    4.4  OTA部署风险。需要梳理出哪些可以部署升级的机型pid，是否都进行过点检，升级策略需要对齐。  ####    4.5 风险释放。在一些特殊场景下，需要对软件进行风险释放，一定要注意相关事项。  [10.OTA部署风险](https://confluence.tclking.com/pages/viewpage.action?pageId=246760823)
