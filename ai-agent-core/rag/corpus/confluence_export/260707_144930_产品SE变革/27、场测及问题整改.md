# 27、场测及问题整改

> pageId: 243464196 | 导出时间: 2026-07-07T14:51:51.780142

角色A更新时间
 
沟通对象场测人员、产品开发经理、测试leader、硬件工程师、合作团队软件接口人具体内容与产品开发经理确认场测计划，输出符合场测要求的软件，保证当地ATV、DTV、网络信号兼容、设备兼容等问题的及时解决。操作方法
1、哪些项目需要进行场测：基础NPI项目和更换tuner、demod的项目；

2、与产品开发经理确认场测计划，包括场测人员、时间路线、测试用例（包含了各区域的场测时间、测试差异等）；

3、输出符合场测要求的软件，并提前安排SQA使用现有的相关码流（各区域积累的场测码流等）进行测试，提前暴露并整改问题；

4、梳理现场反馈的场测问题，组织相关专业工程师进行分析与整改，对于无法在国内复现的问题，特别是严重和概率性问题，应该集中资源进行攻关解决，并在当地进行问题回归测试，且要安排SQA制定相关测试用例；

5、定期输出改善软件，安排SQA对其他可在国内复现的场测问题进行回归测试；

6、遇到典型问题，可要求场测人员录制相关码流或者购买相关设备（码流卡、录制盒、DVD、机顶盒等设备），并将相关资料和设备录入SQA的测试用例资源库。
