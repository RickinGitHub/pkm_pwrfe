# 2.3.3 分支生命周期管理

> pageId: 253024643 | 导出时间: 2026-07-07T14:53:25.045385

| 标题 | **分支生命周期管理** |
| --- | --- |
| 状态 | 已评审 |
| 角色 | A |
| 更新时间 |   |
| 沟通对象 | 领域SE |
| 具体内容 | 看守主干和量产分支的生命周期 |
| 操作方法 | 分支停用：     1、分支停用判定：        ** 项目已经处于维护阶段，并且分支有完成回归主干操作**；      2、停用分支被激活认定：         a.代码提交笔数 >= **5**;         b.量产分支在当月输出版本>=3（版本集成测试任务 jira单/整机测试任务）;          以上两点同时存在，就判断停用量产分支被激活，重新认定为使用中的量产分支      3、分支停用，认定可以不再使用，需要请 @钟小燕在计划时间内关闭代码提交权限 |
| 标准与原则 | - 原则一：同一个TROM版本共用一个产品主干； - 原则二：产品主干按照版本火车规范在指定机芯上过点成功后，拉出量产分支； - 原则三：对于NPI项目周期满足产品主干版本火车过点周期时(T量产时间-T火车过点 >= 1Month），该NPI项目在产品主干上开发，跟随产品主干过点释放；否则，该NPI项目在量产分支上开发，所有量产分支上提交的代码按照三分支管理原则，确保与产品主干同步； - 原则四：当新的量产分支拉出后，老的量产分支上的所有项目已经量产，则关闭该量产分支权限，如果老的量产分支回新量产分支，则关闭该量产分支权限； - 原则五：量产分支生命周期不能超过3个月； - 原则六：量产分支关闭权限，需要重新启用，需要部门长重新审批；版本释放后，及时申请关闭；（超过1个月纳入量产分支统计） |
| 适用项目阶段 | 所有 |
