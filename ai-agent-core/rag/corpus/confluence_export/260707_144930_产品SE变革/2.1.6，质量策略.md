# 2.1.6，质量策略

> pageId: 245932307 | 导出时间: 2026-07-07T14:53:22.020188

| **标题** | **质量策略** |
| --- | --- |
| 项目阶段 | 项目启动 |
| 输入 | 立项需求、需求规格书 |
| 沟通对象 | 质量SE、SPM、模块SE |
| 具体内容 | 项目整体质量策略 |
| 操作方法 | 1.  理解项目价值和需求定义：需要充分了解项目价值（项目目标），明确需求定义；  2. 制定项目质量目标，根据项目立项价值和需求，制定项目质量目标，包括总体目标和过程目标，注意，这些目标应该是可衡量和可验证的。      2.1 总体目标是对软件项目整体质量目标的描述，通常与项目价值和需求相关，为软件项目提供了明确的方向和目标，是项目成功的关键所在。         释放目标的制定，可从以下方面思考：   1.   - 支持项目价值，如产品卖点达成率，需要支撑产品竞争力 2. 关键质量特性，通常有软件产品成熟（DI值），稳定性、性能、UXI体验 3. 在线质量指标：如死机改善率、关键进程崩溃改善率、应用异常改善率 4. 用户态指标：如NPS、全年故障率       2.2 过程目标则是为了实现总体目标而设定的具体目标。过程目标在软件项目中各个特定过程或活动中制定的，以确保在实施过程中达到预期的质量水平。          过程目标的制定，可以从以下方面思考：   1.   - NPI过程关键活动的完成情况，如需求按时上车率，节点通过率 2. 关键质量指标阶段性完成情况，如SR4ADI值，SR4A 稳定性得分 3. 关键质量保障措施的运作情况，如流水线问题发现率、版本有效提测        总体目标和过程目标的各项需要确认责任人，并与责任人达成一致。    3. 制定项目质量策略，包括总体质量策略、目标达成策略等；      3.1 总体质量策略是对软件项目整体质量管理的指导方针和方法的描述，提供了项目团队在实施过程中的方向和引导，以确保整体质量目标的达成。      3.2 目标达成策略则是为了实现总体质量目标而制定的具体策略和方法      3.3 产品SE需要从开发侧，输出项目质量策略，可以从以下方面进行思考：   1.   - 如何支撑质量目标，含释放目标和过程目标 2.  关键抓手（重要模块）是什么  3. 关键质量特性（DI、性能、稳定性、UXI）目标如何达成 4. 软件关键质量保障活动，包括分支管理、流水线、需求管控、SOC代码质量     4. 根据质量目标，制定度量指标和监控机制，定期进行查看，对出现偏差的进行干预    5. 质量策略是一个持续改进的过程，需要根据度量指标和监控结果，及时调整和改进策略，以完成质量目标 |
| 标准与原则 | <1>保证项目全过程的质量得到管理和控制 |
| 指导文件 | 参考《[灵悉OS质量保障计划](https://confluence.tclking.com/pages/viewpage.action?pageId=222712261)》 |
| 适用项目范围 | S、A、B |
| 交付件 | 《质量策略》 |
