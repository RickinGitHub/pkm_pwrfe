# 46、源代码管理

> pageId: 243464228 | 导出时间: 2026-07-07T14:52:25.362335

角色R更新时间
 
沟通对象配置工程师，产品组组长具体内容产品开发及维护阶段，对源代码进行管理操作方法
1，配置工程师给软件leader及各产品组组长分配产品及中间件读权限，便于软件leader查看修改记录。

2、配置工程师只给软件leader及各产品组组长分配特定模块的读写权限，禁止将整个工程的读写权限分配给软件leader及各产品组组长，具体模块如下：

    

| 目录/文件 | 描述 |
| --- | --- |
| tclconfig | PID及屏参配置目录，因项目需求，软件leader需要增加PID或修改屏参配置 |
| update_apk.sh | 集成应用脚本，目前大部分项目的应用集成是软件leader负责，经常需要修改此脚本 |
| device.mk | 集成应用MK文件，系统编译时会将此文件描述的应用集成到版本中去，增减应用时需要修改此文件 |
| /tcl/makefile/Esticker/ | 电子卡牌配置文件夹，不同区域对电子卡牌显示的内容有要求，软件leader在配置PID时需要修改此文件夹 |
| /tcl/makefile | makefile配置文件夹，大部分应用提供的MK文件不满足本平台的要求，软件leader集成应用时需要修改MK文件，并将修改后的MK文件存放在此目录 |
| /tcl/app | 内置应用存放目录，PrimeVideo，Netflix之类的应用存放在此，软件leader会更新或增减存在此目录下的应用 |

3、软件leader上传代码前需要进行同行评审或找组长评审，评审通过后才可以上传
标准与原则1，只修改允许修改的目录及文件2，上传代码前需进行代码评审适用项目阶段
LR、TR、PR、IP、MP
适用项目范围S、A、B、C、D交付件
