# 3、评估报告和计划

> pageId: 243464170 | 导出时间: 2026-07-07T14:51:32.262210

角色R更新时间
 
沟通对象软件开发经理，软件项目经理，各专业负责人，产品开发经理，软件认证经理具体内容根据需求规格说明书中各项功能，输出评估报告，评估报告包括工作量、WBS、MileStone、风险以及规避措施、版本规划、测试计划、认证计划、场测计划等操作方法

1. 根据需求规格说明书（FDS等），组织各专业负责人评估各项功能的工作量；
2. 根据整机计划，结合人力投入预估数据制定《软件开发计划》，该计划由软件项目经理主导制定并输出，软件Leader参与制定；
3. 梳理出风险点，包括重大功能完成节点、认证完成节点、硬件资源（包括开发板、外设、样机需求等）、软件内部资源、SOC人力资源等，由软件项目经理协调；
4. 产品开发经理提出项目认证需求（需求包括认证内容、版本、时间要求等），软件项目经理、软件认证经理、软件开发经理、软件Leader和认证开发Owner制定认证开发测试计划；
5. 根据整机软件计划，与产品开发经理、硬件工程师、软件项目经理制定场测计划，计划要考虑的因素如下：a、不同区域根据量产时间排场测计划，约定提前准备好场测样机的时间；b、保证样机和硬件没有问题的情况下，可以约定先于场测人员寄送样机至场测当地的时间节点；c、若当地TQC有条件的情况，可以考虑在计划中增加提前摸底的测试；d、最终有软件项目经理输出的整机软件开发计划包含了场测计划；
6. 与SQA测试Leader、软件开发经理、软件项目经理根据软件开发计划制定《测试计划》；
7. 汇总上述步骤的输出物，结合项目启动评估的报告，整理输出《软件评估报告》并签字；
8. 软件项目经理负责跟进计划，对于延迟完成的功能上报风险、组织攻关或者评估修改计划。
