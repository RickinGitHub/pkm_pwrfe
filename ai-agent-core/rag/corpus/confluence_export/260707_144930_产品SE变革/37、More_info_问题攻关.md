# 37、More info 问题攻关

> pageId: 243464212 | 导出时间: 2026-07-07T14:52:09.018402

角色A更新时间
 
沟通对象SQA测试Leader、产品开发经理、QPL、模块Owner具体内容项目IP前2个月理清并清理掉遗留的more info问题，避免已发现隐患流到用户端操作方法
1、项目IP前2个月，与软件项目经理、SQA测试Leader梳理mantis/Jira上项目遗留的more info问题，确定more info问题的测试计划，并与问题对应的Owner确认解决计划；

2、对于梳理出来的重点、低概率且需要进行模拟突击的问题，组织SQA测试Leader、问题对应的Owner制定问题模拟计划，然后与SQA测试Leader协调人力执行模拟突击，模拟到问题后按照《[SQA提报问题现场确认](https://confluence.tclking.com/pages/viewpage.action?pageId=14861318)》 指引及时通知对应Owner现场分析定位；

3、对于无法模拟到的more info问题，需要根据《[BUG缺陷分类和处理流程](https://confluence.tclking.com/pages/viewpage.action?pageId=15257065)》中more info问题处理流程，在该类问题变更状态为SCCB时，及时组织产品开发经理、QPL、SQA测试Leader等进行评审，直至问题关闭；
标准与原则问题处理流程应该遵循《[BUG缺陷分类和处理流程](https://confluence.tclking.com/pages/viewpage.action?pageId=15257065)》中more info问题处理流程适用项目阶段
TR、PR、IP
适用项目范围S、A、B交付件
