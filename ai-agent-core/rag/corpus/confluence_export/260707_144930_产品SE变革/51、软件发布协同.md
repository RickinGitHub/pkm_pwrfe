# 51、软件发布协同

> pageId: 243464234 | 导出时间: 2026-07-07T14:52:30.970729

角色A更新时间
 
沟通对象各区域软件Leader具体内容发布计划共享，减小版本的发布操作方法
1、平台软件leader 每周末要 review 下周预计要发布的版本，确认各区域整机软件需求计划，尽量做到版本合并，并登记在《软件发布大盘表》上（软件发布计划可以根据具据项目大盘表，新功能导入等需求来规划）
2、各区软件leader发布版本时，要填写《软件发布版本登记》,避免不同区域发软件时出现版本冲突
3、各区软件leader在计划外发布版本时，要在群里告知其他区域leader，确认其它区域是否也有发软件需求，减少软件发布版本
