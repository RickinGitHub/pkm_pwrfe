# 4.1.1 需求可行性评估

> pageId: 246759158 | 导出时间: 2026-07-07T14:50:34.784229

| **标题** | 需求可行性评估 |
| --- | --- |
| 项目阶段 | SR1 |
| 角色 | A |
| 输入 | 《软件需求规格说明书》什么形式？内容？ |
| 沟通对象 | 需求工程师、整机产品开发经理、硬件工程师、SOC、涉及专业工程师 |
| 具体内容 | 主导软件团队进行需求可行性评估，输出《需求可行性评估报告》，包括可行性、工作量、风险 |
| 操作方法 | 1、需求工程师根据《软件需求规格说明书》组织相关软件团队（包括产品SE、模块owner、 模块SE、软件项目经理、硬件工程师）进行评估。  2、各模块owner根据需求进行专业评估、输出方案、可行性、工作量、风险；  3、各模块SE评审模块owner的可行性评估报告  4、产品SE评审各模块的可行性评估报告，并形成整体可行性评估报告  5、需求工程师依据《需求可行性评估报告》，和整机开发经理达成一致，由软件部部长签批。 |
| 标准与原则 |  |
| 指导文件 | 《需求可行性评估指引》、《需求可行性评估报告模板》 |
| 适用项目范围 | S,A,B项目 |
| 交付件 | 《需求可行性评估报告》 |
