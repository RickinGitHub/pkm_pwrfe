# 2.1.1，代码合入

> pageId: 245932269 | 导出时间: 2026-07-07T14:53:19.533766

| **标题** | **代码合入** |
| --- | --- |
| 项目阶段 | 项目各阶段 |
| 输入 | 代码提交 |
| 沟通对象 | 功能owner，模块SE，SOC，CIE |
| 具体内容 | 量产必解问题的修改和对应JIRA号 |
| 操作方法 | 量产分支的上游分支:主干分支 1、准备提交的代码--需要保证上游已经提交，且功能测试正常、稳定 2、使用Cherr-Pick方式将上游的提交同量产确保与上游分支Changeld一致  3、保证自测通过，符合编程规范  4、自测报告:包括功能测试，稳定性测试 5、启动预编译，编译通过后，方可启动审核  6、 评审人进行评审，提出问题 7、代码提交人针对所提问题进行修改，修改后和评审人员确认无误后标记解决，问题全部鲜决后方可进入代码合入阶段 8、模块SE对方案和修改负责，入库前需打加分+2  9、评审人员评审通过后，产品SE再submit入库  10、合入之后功能owner需要使用合入后的代码再次验证，确保功能正常 |
| 标准与原则 | <1>量产分支上游分支: 主干分支 <2>量产分支原则上禁止合入新的功能，只能合入量产必解问题的修改，特殊情况需[产品SE] 评审。 <3>量产分支只有[产品 SE] 有权限 submit |

| 适用项目范围 | S、A、B、C、D |
