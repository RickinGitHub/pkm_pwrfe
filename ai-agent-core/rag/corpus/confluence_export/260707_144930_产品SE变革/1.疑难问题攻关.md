| **标题** | 疑难问题攻关 |
| --- | --- |
| **状态** | 已评审 |
| **角色** | A |
| **更新时间** |   |
| **沟通对象** | 各软件模块Owner、模块SE、SEG、SQA |
| **具体内容** | 召集相关模块owner进行问题分析和难点梳理，制定行动项和推进计划 |
| **操作方法** | - **疑难问题攻关范围**  1）已投入分析但短期无有效解决方案的问题  2）跨模块跨专业，无人主导分析的问题  3）模块owner提出需要协助的问题   - **疑难攻关手册**  1）针对问题现象、现有分析进展，以及相关专业知识初步判断问题方向；  2)  产品SE主导问题讨论与攻关，主要参与和确认以下几方面：        a. 问题复现情况和已进行的分析情况梳理；        b. 问题分析定位和难点讨论；        c. 制定行动项和责任人及完成时间期限；        如无有效方案，要求模块SE上升SEG（），由SEG主导攻关  3）SPM整理会议记录，邮件给各参会人员和相关资源部门部门长；  4）SPM跟进各行动项进展，对进展缓慢项进行风险预警；  5）根据行动项反馈结论及时分析或调整计划，直至闭环 |

| **交付件** | 按中心要求输出会议纪要，问题分析报告 |
