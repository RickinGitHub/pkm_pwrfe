# 1. 技术可行性评估

> pageId: 247458485 | 导出时间: 2026-07-07T14:53:18.092237

| 标题 | 技术可行性评估 |
| --- | --- |
| 状态 | 待评审 |
| 角色 | A |
| 更新时间 |   |
| 沟通对象 | 相关开发人员，系统SE、模块SE、产品SE、SPM |
| 具体内容 | 针对重点、难点功能需求，组织方案的设计和评审 |
| 操作方法 | 1、主导对需求初步分析，和需求提出方沟通，明确需求规格定义  2、主导确定需求涉及的相关模块。  3、拉通相关模块对应的模块SE或者模块OWNER，对需求涉及的关键技术点进行梳理，输出关键技术点列表  4、针对每个关键技术点，相关的SE或者模块owner进行可行性分析，并识别风险点，输出可行性分析结论及风险列表  5、确定风险应对策略  6、主导拉通各模块SE，确定整体需求的人力评估  7、主导拉通模块SE，汇总输出可行性分析报告 |
| 标准与原则 | 1. 需求分析时要考虑影响范围和其他依赖  2. 需要考虑对现有功能的兼容性；涉及多机芯需求，要考虑是否通用，是否存在机芯差异，是否可以兼容OTA升级  3. 需要描述可能出现的风险以及应对措施。  4. 需要考虑是否对特定产品有影响：如空间占用，pid使用。  5. 需要描述技术的优势、劣势以及人力评估 |

| 适用项目范围 | S、A、B、C |
