# 18、影响SQA/DQA测试的问题优先级梳理

> pageId: 243464186 | 导出时间: 2026-07-07T14:51:43.405003

角色A更新时间
 
沟通对象SQA测试Leader，硬件工程师，产品开发经理，软件项目经理，软件模块owner具体内容明确影响SQA和DQA测试问题的优先级，并推动解决操作方法1、软件发布给SQA测试前，应该先跑产品软件的自测用例，主要确认是否有开机异常、明显死机或重启问题、主要功能是否缺或有较严重问题，如果发现有此类问题（或者在SQA测试中发现），应该要根据测试目标确认是否要先暂停发布和测试，优先解决此问题后再重新发布软件测试，以此来保证发布的软件可以按照测试计划和策略的执行不受此类严重问题的影响；
2、项目中发现有阻碍硬件、PQ、AQ、DQA等测试问题，应该与相关工程师沟通问题具体情况，看是否有临时方案帮助测试工作继续完成，如果没有临时方案，则应该优先推动解决此类问题，避免影响整机测试的进度；
3、根据整体开发计划，梳理影响测试的问题点，制定解决计划，由软件项目经理协助推进解决。标准与原则
适用项目阶段 LR，TR，PR适用项目范围S、A、B、C、D交付件
