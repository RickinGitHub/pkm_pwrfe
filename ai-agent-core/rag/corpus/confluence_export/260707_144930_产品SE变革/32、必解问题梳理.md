# 32、必解问题梳理

> pageId: 243464203 | 导出时间: 2026-07-07T14:52:02.025104

角色R更新时间
 
沟通对象产品开发经理、软件开发经理、软件项目经理、部门长、SQA测试Leader具体内容梳理必解问题操作方法
1、软件Leader遍历mantis上的问题，熟悉各问题的基本情况及进展；

2、软件Leader组织SQA测试Leaer，产品开发经理，软件开发经理、产品和SQA部门长开展必解问题梳理会议；

3、会议评审过程中，按以下标准梳理必解问题：

    a、问题属于项目需求；

    b、系统问题且影响用户使用；

    c、非低概率的体验性问题；

    d、确定短时间内可解决的问题；

4、给必解问题打上必解标签，由软件项目经理制定解决计划，推进必解问题迅速收敛，并在IP前收敛所有必解问题；
标准与原则影响项目释放的问题必解适用项目阶段LR、TR、PR适用项目范围S、A、B交付件必解问题表备注**必解问题标准另行制定**[返回目录](https://confluence.tclking.com/pages/viewpage.action?pageId=9052451)
