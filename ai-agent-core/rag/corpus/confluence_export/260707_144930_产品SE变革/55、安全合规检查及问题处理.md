# 55、安全合规检查及问题处理

> pageId: 243464241 | 导出时间: 2026-07-07T14:52:38.561148

角色[编写中](https://confluence.tclking.com/pages/editpage.action?pageId=14861363)A更新时间
 
沟通对象安全合规测试人员，安全合规经理，软件项目管理具体内容安全合规检查及问题处理操作方法
分两部分：一个是新方案及软件发布：
一，新方案SE评审：
1、新方案需求定下来后，各团队提供设计文档；

2、由产品开发经理或者软件leader组织各模块团队安排SE评审。

3，SE评审通过了，才能真正导到产品上。

二，新软件SE评审：
1、新软件发布后，软件leader通知安全合规测试人员对软件进行安全合规测试

2、软件合规测试报告发出后，软件leader通知安全合规经理过滤无效问题，确认那些问题需要确认或修改

3、软件leader将需要确认和修改的问题分配给对应的模块或应用维护人员，并通知软件项目管理跟进处理进度。

4、软件leader根据问题的处理情况，组织安全合规经理对存在的安全合规问题进行评审，判断安全合规问题是否影响软件的释放
