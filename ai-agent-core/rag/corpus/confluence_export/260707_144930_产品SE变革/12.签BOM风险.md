| 标题 | 签BOM风险 |
| --- | --- |
| 状态 | 待评审 |
| 角色 | A |
| 更新时间 |   |
| 沟通对象 | #### 硬件工程师，PQ，PMO |
| 具体内容 | ### 把控签bom风险，特别在新机型，新派生立项的时候需要进行签bom，要注意hid，panel，pid，sid等配置，确认无误。 |
| 操作方法 | 1.当工厂发起BOM签署时一定要注意以下几点：     1.1： 硬件用的HID是多少，是否已经上code     1.2：软件所需要的SID是多少，是否所有机型共用SID。（机型能否共用SID参考依据是是否共用数据库，是否共用app json等文件）     1.3 ：通过算法计算出pid（需要注意目前pid是用新的算法还是老的，算法不一样所计算出来的pid也不一样）     1.4：panel 是什么规格，一定要备注清楚方便后续查询     1.5所签的bom需要更新到项目文档里面（各个项目应该有维护）  2.在签bom过程中一定要注意是否是本项目机型，特别是派生机型需要跟需求经理确认是否已经立项。签bom过程中任何可疑点需要及时找相关owner对齐。 |
| 标准与原则 | 按软件中心项目管理模板输出 |
