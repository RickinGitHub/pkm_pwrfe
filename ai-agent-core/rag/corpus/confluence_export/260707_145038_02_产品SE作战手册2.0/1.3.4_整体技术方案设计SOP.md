# **整体技术方案设计SOP**

### 1. 前置准备：全面了解项目需求与硬件规格

在撰写《XXX软件项目整体方案设计》报告前，请先完成以下信息收集与确认：

- 
**项目宏观信息**：项目背景、项目目标、产品定位、新品规划的核心卖点。

- 
**器件与规格**：

- 
IC及其它器件的规格及对标情况。

- 
需求规格说明书 & XX产品FDS。

- 
**硬件方案细节**（整机，包括主板、屏幕、外设等）：

- 
各外围方案的型号、控制方式。

- 
硬件特殊需求等规格。

### 2. SOC方案评估：掌握能力并识别差距

产品SE需执行以下评估动作：

1. 
**掌握SOC方案规格**：熟悉本项目所用SOC方案的架构与能力，包括：

- 
CPU、GPU、APU算力

2. 
内存架构方案等

3. 
**与上一代产品对标**：明确差异点，判断SOC方案能否满足关键功能和技术方案需求。

4. 
**制定评估计划**：若部分能力需进一步验证，请列出评估项并制定详细评估计划（含时间与责任人）。

### 3. 报告撰写：聚焦差异、核心功能与实现思路

《XXX软件项目整体方案设计》报告中，必须清晰呈现以下内容：

**差异需求/功能设计方案**：与上一代产品（或项目）相比，本次新增或变更的需求/功能如何设计。

- 
**核心功能设计方案**：对关键功能进行详细设计。

- 
**实现方法与思路**：说明达成上述功能/需求所采用的技术方法、设计思路或算法流程。

### 4. 报告核心要点：技术抓手、目标与风险应对

报告中需单独列出以下内容：

- 
**技术核心抓手**：本方案最重要的技术着力点。

- **核心抓手目标及应对策略**：每个抓手要达成的具体目标，以及为实现目标采取的策略。
- 
**风险点与应对措施**：识别潜在技术/进度风险，并逐条给出可落地的应对方案。

### 5. 评审规范：按中心要求执行

按照中心设计方案评审规范进行评审，重点注意：

| 类别 | 具体要求 |
| --- | --- |
| **评审级别** | S类项目的整体方案设计，需经**部门长和总监**评审。 |
| **评审结论** | 必须包含：**具体工作项、责任人、完成时间节点**。软件项目经理负责跟进闭环。 |
| **分阶段交付** | 功能复杂时，可组织与BA评估是否分阶段实现和交付。 |
| **质量属性检查** | 设计方案必须考虑：兼容性、异常处理、稳定性、性能、用户体验。 |
