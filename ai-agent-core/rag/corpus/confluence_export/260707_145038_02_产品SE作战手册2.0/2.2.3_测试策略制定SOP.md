# 2.2.3 测试策略制定SOP

> pageId: 588380500 | 导出时间: 2026-07-07T14:53:32.749601

# **SOP简介：**

**文档主要内容：** 软件发布后协助VPM制定项目软件的测试策略和测试计划

**文档适用角色：**大产品SE（机芯软件负责人）& 小产品SE（派生软件负责人）、模块开发负责人

**适用项目阶段：**SR3，SR4，SR5，SR6

**环境依赖：**

**相关内容链接：**测试策略模板V3.1（已发布）

## **1、测试策略制定目的**

制定测试策略的目的是形成一套合适的测试方案，在兼顾项目交付周期的前提下，以质量可靠性为基础保障项目按期交付，同时尽可能在测试阶段一次性充分暴露软件中存在的问题，减少后续反复验收测试的次数，提升整体测试效率。

产品SE是以参与会议的角色进行测试策略的评审。

## **2、****测试策略制定与评审****策略**

1）严格控制回归测试版本的修改导入范围，避免引入改动较大、风险不可控的修改，尤其是在时间紧急的版本中，应优先保证版本稳定性。

2）明确问题的影响范围及对应的测试维度，特别是针对严重问题或影响面较大的问题，需要重点分析并制定针对性的测试策略。

3）对于原因或影响范围不明确的问题，应与开发owner充分沟通，确认问题根因、解决方案及影响范围。尤其是涉及封装代码的修改（如SOC、KO等），需要重点关注其潜在影响。

4）积极参与VPM组织的版本测试策略评审，确保测试方案在多方协同下达成一致。

5）在评审过程中重点关注并决策以下内容：

（1）测试策略是否能够覆盖所有修改可能带来的影响范围，避免出现漏测情况；

（2）测试周期是否能够满足项目进度要求，并预留足够时间验证测试过程中发现的新问题；

（3）是否存在测试资源浪费的情况，避免过度测试与本次修改无关的内容；

（4）对于存在不确定性的内容，建议及时邀请更有经验的产品SE或上级领导参与评估和决策。

## **3、****测试策略制定流程**

1）VPM依据《NPI通用测试策略规范 V2.2（已发布）》的要求，结合不同项目等级（S/A/B/C/D）及项目所处阶段，制定相匹配的测试策略。

2）VPM在明确测试策略后，结合测试周期、测试人力等实际情况，统筹并落实相应的测试设备及测试资源。

3）VPM负责组织测试策略评审，评审成员至少包括：产品SE、领域SE、质量SE和SPM。

4）最终需形成并输出相关交付文档，包括：《测试策略》《测试计划》以及《评审纪要》。
